Introduction aux Réseaux

Définitions

BIT = Binary unIT (unité binaire :).

Internet = réseau de réseaux.

Cable ethernet = cable de paires torsadées (twisted pair).

Rootage = trouver le bon chemin (sur plusieurs possibles comme sur Maps).

Forwarding = guider (dire chaque sortie de rond-point :).

Packet-switching

Utilisé sur internet :

  • Store-and-forward (si noeud dispo) :
    • Délai de transmission de paquet : prend L/R secondes à transmettre avec des paquets de L bits à R bps.
    • Store and forward : introduit de nouveaux délais.
  • Queueing (si noeud pas dispo) :
    • Les paquets sont mis en liste d'attente (queue) en attente de transmission.
    • S'il est quand même saturé, il ignore les paquets entrants et ils sont perdus !

Internet

Un réseau de réseaux complexe influencé par l'économie et les politiques nationales.

Performances

  • Les pertes de données se font lorsque la file d'attente est pleine.
  • Définitions :
    • Temps de d'attente (queueing delay) = Longueur du paquet (bits) * Taux d'arrivée des paquets / Taux de transmission du lien (bps).
    • Temps de nodal processing : Temps qu'il faut pour le noeud de déterminer d'où vient le paquet, où il va, s'il y a une erreur...
    • Temps de transmission = Longueur des paquets (bits) / Vitesse de transmission du lien physique (bps).
    • Temps de propagation = Longueur du lien physique (m) / Vitesse de propagation (~2x10^8 m/s).

Sécurité du réseau

  • Problématique récente (25-30 ans) avec le déploiement d'internet.
  • Protocoles mis en place pour chaque type de transmission (communications, liens privés, informations...).
  • Packet-sniffing : Écoute des communications (en se mettant sur le même câble, réseau wifi...).
  • IP spoofing : Injection de paquets en utilisant une fausse adresse (pur espionnage ça).
  • Denial of Service : Rend la bande passante indisponible pour du trafic normal sur un serveur.

Modèle en "couches" de protocoles

Pourquoi faire des couches ? Plus efficace : on sait qui fait quoi en gros.

  • Modèle ISO/OSI
    • Physique : Transmission des signaux informatiques sur deux éléments (signaux électriques).
    • Lien : Transfert du signal entre deux éléments voisins (Ethernet, 802.11(Wifi), PPP).
    • Réseau : Identification de l'émetteur/recepteur sur le réseau (IP).
    • Transport : Transmission des communications sur le réseau (TCP, UDP).
    • Application : Tout ce qui utilise les paquets reçus pour en faire autre chose (HTTP, IMAP, SMTP, DNS).
