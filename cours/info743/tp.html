TP1 - Introduction aux sockets

Création d'un socket (Java)

Coté serveur :

// Création du socket serveur    
ServerSocket server = new ServerSocket(port); // Création d'un serveur (port: port du serveur)

// Création d'un socket client sur le serveur
Socket socket = server.accept(); // Attente de connexion d'un client

Coté client :

// Création du socket client
Socket socket = new Socket(hostname, port); // Connexion au serveur (hostname: nom du serveur, port: port du serveur)

La méthode accept() est bloquante, elle attend qu'un client se connecte.

TP2 - Wireshark

Capture HTTP

On peut filtrer les paquets capturés en utilisant le filtre http pour ne voir que les requêtes HTTP.

Dans la capture, on peut voir les entêtes de la requête HTTP (méthode, URL, version, etc.) et les données envoyées.

Dans ces entêtes, on peut voir la version HTTP, l'IP client (10.7.*.*), l'IP serveur et le port utilisé (80 pour HTTP, 443 pour HTTPS).
On peut aussi voir les langues acceptées :

Accept-Language: fr,fr-FR;q=0.9,en;q=0.8,en-GB;q=0.7,en-US;q=0.6,es-ES;q=0.5,es;q=0.4\r\n

Si le site est déjà en cache, le navigateur envoie une requête GET avec un entête If-Modified-Since pour vérifier si la version en cache est à jour. Si c'est le cas, le serveur renvoie un code 304 Not Modified sans renvoyer le contenu. Sinon, il renvoie le contenu.

Quand il y a des images, le navigateur envoie une requête GET pour chaque image (en parallèle) pour les charger. Les requêtes sont envoyées vers le serveur de l'image.

Si on tente de se connecter à un site avec une connexion, on reçoit le code 401 Unauthorized et le serveur demande une authentification. Après ça, on reçoit un code 200 OK.
Cependant, en HTTP, les identfiants sont visibles dans Credentials et donc non sécurisés.

TP3 - Configuration de réseau et routage

Configuration d'une interface réseau

Sur Linux, on peut configurer une interface réseau avec la commande ifconfig :

ifconfig eth0 10.0.0.1 netmask 255.255.255.0 broadcast 10.0.0.255 up

Ici, on configure l'interface eth0 avec :

  • l'adresse IP 10.0.0.1,
  • le masque de sous-réseau 255.255.255.0,
  • l'adresse de broadcast 10.0.0.255.

On active ensuite l'interface avec up.

Si deux machines sont sur le même réseau (même masque de sous-réseau), elles peuvent communiquer directement (avec ping par exemple).

Configuration d'une route

On peut afficher les routes avec la commande :

route -n

Résultat :

192.168.141.0/24 dev eth0 proto kernel scope link src 192.168.141.101

On peut ajouter une route avec la commande :

route add -net ip_cible netmask 255.255.255.255 gw ip_passerelle eth0

On ne peut ping une machine que si on a une route vers cette machine. Si on n'a pas de route, on reçoit un message Destination Host Unreachable.

Commandes supplémentaires

  • Pour afficher les tables ARP (Liste des adresses résolues)

    arp -an

  • Pour afficher les voisins IPv4 (Liste des adresses IP voisines, pas certain mdr)

    ip -4 neigh

    → En gros le serveur enregistre les machines avec lesquelles il a déjà communiqué pour ne pas avoir à résoudre l'adresse MAC à chaque fois.

  • Pour faire une capture sur Wireshark sous Linux

    tcpdump (-i eth0) (-s taille max) -w capture.cap
:: CTRL+C pour arrêter la capture

TP4 - Firewall

Configuration des interfaces d'une machine (pc1.startup)
(Idem pour webserver et sqlserver)

ifconfig eth0 1.2.3.2 netmask 255.255.255.0 up
route add default gw 1.2.3.1 dev eth0

Configuration des interfaces du firewall (firewall.startup)

ifconfig eth0 1.2.3.1 netmask 255.255.255.0 up
ifconfig eth1 5.5.5.1 netmask 255.255.255.0 up
ifconfig eth2 5.5.6.8 netmask 255.255.255.0 up

Configuration du lab kathara (lab.conf)

pc1[0]="A"
firewall[0]="A"
firewall[1]="B"
firewall[2]="C"
webserver[0]="B"
sqlserver[0]="C"

Configuration du firewall (firewall.startup)

Pour voir la liste des règles :

iptables -L

Blocage des paquets ICMP entre deux machines

iptables -A FORWARD -p icmp -s <adresse source> -d <adresse destination> -j DROP

Retrait de la règle de blocage :

iptables -D FORWARD -p icmp -s <adresse source> -d <adresse destination> -j DROP
